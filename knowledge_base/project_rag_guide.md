# 项目知识库说明

这个目录会被项目内置的 RAG 自动扫描，用来回答“项目内部资料、业务规则、私有文档、FAQ、产品说明”等问题。

## 当前项目能力

- 后端是一个基于 LangChain 的 Agent。
- 已支持 OpenAI / Anthropic 双模型切换。
- 已支持天气工具调用。
- 已支持 Tavily 实时网页搜索。
- 已支持通过 `api_server.py` 提供 `POST /api/chat` 接口。
- 前端位于 `next-ui/`，当前通过 `NEXT_PUBLIC_API_BASE_URL` 调用 Python API。

## RAG 工作方式

- 服务启动后会自动扫描 `knowledge_base/` 目录下的文档。
- 支持的文件类型包括：`md`、`txt`、`rst`、`json`、`csv`。
- 文档会被切分为多个 chunk。
- 用户提问时，Agent 可以调用本地知识库检索工具，取回最相关的片段后再生成回答。

## 如何扩展知识库

1. 把业务资料放进这个目录，建议优先使用 `Markdown` 或 `TXT`。
2. 一个主题一个文件，文件名尽量语义化，例如 `refund_policy.md`、`deployment_notes.md`。
3. 文档中尽量加入清晰标题、分节和关键词，方便检索命中。
4. 更新文件后不需要手动建索引，下一次请求会自动检测变化并刷新索引。

## 推荐内容类型

- 产品功能说明
- 售后和客服 FAQ
- 内部流程文档
- 部署说明
- Prompt 规范
- 数据口径说明

## 使用建议

- 如果问题属于公开实时信息，仍然应优先使用网页搜索工具。
- 如果问题属于当前项目或你的私有资料，优先使用本地知识库检索。
